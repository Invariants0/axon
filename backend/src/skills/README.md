# Skill System Overview

## Architecture

The skill system enables modular, composable task execution through:

### Key Components

1. **SkillRegistry** (`src/skills/registry.py`)
   - Auto-discovers all skills in `core_skills/` and `generated_skills/` directories
   - Builds `SkillDefinition` objects with metadata (name, description, parameters, version)
   - Requires each skill module to export a `SKILL` dict and async `execute()` function

2. **SkillExecutor** (`src/skills/executor.py`)
   - Validates payload parameters against skill specs
   - Executes skills and wraps results with metadata
   - Handles both async and sync execute functions

3. **Skill Modules**
   - Located in `core_skills/` (permanent, built-in) or `generated_skills/` (dynamic, LLM-generated)
   - Must export: `SKILL` dict and `execute(payload: dict) -> dict` function
   - Can be registered dynamically at runtime

## Skill Lifecycle

```
┌─────────────────┐
│  SkillRegistry  │ ← Auto-discovers modules in core_skills/ and generated_skills/
└────────┬────────┘
         │
         ↓
┌─────────────────────────────┐
│  SkillDefinition (metadata) │ ← name, description, parameters, version
└────────┬────────────────────┘
         │
         ↓
┌──────────────────────────────────────┐
│  SkillExecutor.execute()             │
│  1. Validate payload parameters      │
│  2. Call skill.execute(payload)      │
│  3. Wrap result with metadata        │
└──────────────────────────────────────┘
```

## Creating a New Skill

1. Create `backend/src/skills/core_skills/my_skill.py`:

```python
SKILL = {
    "name": "my_skill",
    "description": "What this skill does",
    "parameters": {
        "input_key": {"type": "string", "required": True},
    },
    "version": "1.0.0",
}

async def execute(payload: dict) -> dict:
    input_val = payload.get("input_key", "")
    # Your implementation here
    return {"result": input_val}
```

2. SkillRegistry will auto-discover it on next startup
3. Any agent can now call: `skill_executor.execute("my_skill", {...})`

## Notes

- Skills are **pure functions**: no side effects, deterministic output
- Skill execution is **logged** and tracked in the event bus
- Skill parameters are **validated** before execution
- Skills can be **generated** by agents and stored dynamically
- For live web search: update `generated_skills/web_search.py` with real API client
